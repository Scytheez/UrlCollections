# UrlCollections

A clean, minimalist, and lightweight web application built to curate, organize, and quickly navigate bookmarks and essential web resources. 

Live Demo: [scytheez.github.io/UrlCollections](https://scytheez.github.io/UrlCollections/)

---

## 🚀 Features

* **Categorized Lists:** Easily segment resources by purpose, topic, or stack.
* **Minimalist UI:** Clean layout optimized for fast reading and navigation.
* **Responsive Design:** Fully mobile-friendly implementation that scales to any screen size.
* **Static Deployment:** Ready to be hosted anywhere with zero server configuration required.

## 🛠️ Tech Stack

* **HTML5:** Semantic structure for maximum web accessibility.
* **CSS3:** Custom layouts, variables, and responsive styling rules.
* **GitHub Pages:** Automated, free, and secure web hosting pipeline.

## 📦 Getting Started

### Prerequisites
You only need a modern web browser to view the application locally.

### Local Installation
Follow these simple steps to run the application on your computer:

1. Clone this repository down to your local file system:
   ```bash
   git clone https://github.com
   ```
2. Navigate directly into the project directory:
   ```bash
   cd UrlCollections
   ```
3. Open the entry file in your browser:
   * Double-click `index.html` within your file explorer, OR
   * Run it from your terminal (macOS/Linux): `open index.html`

## 🎨 Customization

To add your own bookmarks or modify the collection structures:
1. Open the project inside your favorite IDE (e.g., VS Code).
2. Locate the main structure files inside the root directory.
3. Edit the anchor elements (`<a>`) using standard URL formats:
   ```html
   <a href="https://example.com" target="_blank" rel="noopener noreferrer">Your Link Title</a>
   ```

## 🌐 Deployment

This project is configured to publish automatically via GitHub Actions or the standard repository configuration:
1. Push your changes back up to your remote GitHub repository server.
2. Go to your repository settings page on GitHub.
3. Select the **Pages** tab on the left sidebar navigation menu.
4. Set the publishing source branch to `main` (or your default branch) and directory to `/root`.
5. Save settings to build your new custom deployment pipeline.
